# Contributing

Thanks for helping improve Video Summary.

## Local setup

```bash
git clone https://github.com/Rexxall/obsidian-video-summary-plugin.git
cd obsidian-video-summary-plugin
npm install
npm run build
```

For active development:

```bash
npm run dev
```

Install the built files into a test vault:

```text
main.js
manifest.json
styles.css
Obsidian Video Summary.json
```

## Before opening a pull request

- Keep changes focused on one behavior or documentation topic.
- Run `npm run build`.
- Do not commit vault data, cache files, cookies, credentials, or personal transcripts.
- If the n8n workflow changes, explain the migration impact.
- If the request payload changes, keep backward compatibility or document the required workflow update.

## Code style

- TypeScript source lives at the repository root and supporting folders.
- `main.js` is generated by esbuild and is included only for Obsidian release assets.
- Keep user-facing copy clear and specific.
- Prefer compatibility with existing payload keys and settings unless a breaking change is intentional.

## Release changes

When preparing a release:

1. Update `package.json` and `manifest.json` to the same semantic version.
2. Update `versions.json` with the minimum supported Obsidian version.
3. Run `npm run build`.
4. Push a tag matching the manifest version, for example `2.1.3`.
